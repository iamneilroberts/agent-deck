# AGENTS.md — instructions for coding agents in this repo

AgentDeck is a self-hosted web app that controls local coding agents (Codex, Claude Code)
from a phone over Tailscale. Read [`docs/architecture.md`](docs/architecture.md) before
making design decisions; read [`docs/adr/0001-adapter-architecture.md`](docs/adr/0001-adapter-architecture.md)
before touching the adapter boundary.

## Current phase

**Phase 0 — spikes.** The only code that exists is `spikes/codex-app-server/`. Do not start
the full server or frontend until the spikes are documented and the normalized event model is
settled. Follow the phase order in `docs/architecture.md` §20.

## Ground rules

- **Faithful transport, never a re-implementation.** Map the agent's own protocol into a
  vendor-neutral event model; do not invent capabilities (e.g. approval scopes) the agent
  does not expose. The Codex protocol hands us `availableDecisions` per approval — use it.
- **Two ID spaces on the Codex wire.** Client request IDs and server→client request IDs are
  independent. Route inbound messages by "has a `method`", not by id. See the spike's
  `transport.ts`.
- **Secrets never leave the host.** No `.env` values or auth tokens to the browser or into a
  committed log. Sanitize protocol captures (home dir → `$HOME`, tokens redacted) before
  writing them to `captured/`. Raw captures are gitignored.
- **Localhost only.** Nothing binds to `0.0.0.0`. Access is Tailscale + a reverse proxy bound
  to the Tailscale interface; the app binds to `127.0.0.1`.
- **Strict TypeScript, no `any`, validate external input.** Vendor-specific types stay inside
  adapter packages behind the neutral `AgentAdapter` interface.
- **Add tests with every behavioral change.** Pure functions (framing, redaction, state
  transitions) are unit-tested; adapter behavior is tested against fake agent processes, not
  paid API calls.

## Protocol regeneration

The Codex app-server protocol types are machine-generated by the installed binary:

```
codex app-server generate-ts --out <dir>
codex app-server generate-json-schema --out <dir>
```

The spike vendors a hand-curated subset in `src/proto.ts` (only the messages it uses),
clearly marked. When bumping the Codex version, regenerate and reconcile.

## Handoff convention (once the app exists)

Durable state lives in Git and `.agent/` files (`HANDOFF.md`, `CURRENT_TASK.md`,
`DECISIONS.md`, `SESSION_LOG.md`) — never in an agent's hidden context. A handoff updates
those files and confirms tree state; it does not translate a context window.
